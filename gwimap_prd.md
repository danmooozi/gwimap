# 👻 귀맵 (가칭) - 심령 스팟 지도 서비스 PRD

## 1. 개요

### 1.1 서비스 정의

사용자들이 제보한 심령 경험, 귀신 목격담, 오싹한 장소 정보를 기반으로  
지도 위에 심령 스팟을 시각화하는 참여형 웹 서비스

### 1.2 목표

- 여름 시즌 트래픽 확보 (시즈널 콘텐츠)
- 사용자 참여 기반 콘텐츠 축적 (UGC)
- “공포 경험 공유 플랫폼”으로 확장 가능성 확보

### 1.3 핵심 가치

- 재미 (Entertainment-first)
- 참여성 (User Generated Content)
- 탐험성 (지도 기반 발견 경험)

---

## 2. 타겟 사용자

### 2.1 주요 타겟

- 공포 콘텐츠를 즐기는 10~30대
- 심령 유튜브 / 공포 썰 / 괴담 콘텐츠 소비자
- 친구들과 “놀 거리” 찾는 사용자

### 2.2 사용 시나리오

- “우리 근처에 귀신 나오는 곳 있나?” → 지도 탐색
- “나 여기 갔는데 진짜 이상했어” → 제보 등록
- “무서운 장소 TOP 리스트 보고 싶다” → 랭킹 확인

---

## 3. 핵심 기능

### 3.1 지도 기반 심령 스팟 탐색

- 지도 위에 심령 스팟 마커 표시
- 확대/축소 시 클러스터링 처리
- 위치 기반 탐색 (현재 위치 중심)

#### 마커 정보

- 스팟 이름 (또는 별칭)
- 공포 등급 (Lv1 ~ Lv5)
- 제보 수
- 대표 이미지

---

### 3.2 심령 스팟 상세 페이지

- 위치 정보 (지도 + 주소)
- 사용자 제보 리스트
- 대표 심령 사진
- 공포 레벨 및 태그

#### 포함 정보

- 👁️ 목격담
- 📸 사진/영상
- ⏰ 발생 시간대
- 😱 공포 유형 (귀신, 소리, 기운 등)

---

### 3.3 사용자 제보 기능 (UGC)

#### 입력 항목

- 위치 선택 (지도 클릭 or 검색)
- 제목
- 경험 내용 (텍스트)
- 사진 첨부 (선택)
- 공포 레벨 선택 (1~5)
- 카테고리 선택 (목격 / 소리 / 느낌 등)

#### 특징

- 익명 제보 가능
- 간단한 UX (빠른 등록)

---

### 3.4 공포 레벨 시스템

- Lv1: 그냥 기분 나쁨
- Lv2: 이상한 느낌
- Lv3: 설명 안 되는 현상
- Lv4: 확실한 목격
- Lv5: 강력한 심령 경험

→ 시각적으로 색상/아이콘 차별화

---

### 3.5 랭킹 / 인기 콘텐츠

- 🔥 이번 주 인기 스팟
- 👻 가장 무서운 장소 TOP 10
- 📍 지역별 인기 스팟

---

### 3.6 검색 및 필터

- 지역 검색
- 공포 레벨 필터
- 카테고리 필터

---

## 4. UX / UI 방향

### 4.1 디자인 컨셉

핵심 디자인 시스템은 프로젝트 루트 -> DESIGN.md 사용

- 다크 모드 기반
- 레트로 공포 느낌 (CRT, 노이즈, VHS 감성)
- 과도한 리얼리즘 X → “귀엽고 오싹한” 방향도 고려 가능

### 4.2 인터랙션

- 마커 hover 시 미세 애니메이션
- 공포 레벨에 따라 떨림 효과
- 특정 시간대(밤)에 UI 변화 가능

### 4.3 화면 구성 및 디자인

1. 메인 화면 - /assets/main_map_screen 참고
2. 스팟 상세 화면 - /assets/detail_spot_screen 참고
3. 스팟 제보 화면 - /assets/request_spot_screen 참고

## 5. 기술 요구사항

### 5.1 프론트엔드

- 지도 라이브러리: (ex. Kakao Map)
- 이미지 업로드 처리
- 마커 클러스터링

### 5.2 백엔드

- 가능하다면 별도 백엔드 구축 없이 serverless 사용 (e.g. firestore, supabase)
- 스팟 / 제보 CRUD API
- 이미지 저장 (S3 등)
- 위치 기반 쿼리 (Geo query)

### 5.3 데이터 구조 (예시)

#### Spot

- id
- lat, lng
- name
- fearLevel
- reportCount

#### Report

- id
- spotId
- content
- imageUrl
- fearLevel
- createdAt

## 6. 정책 및 리스크 관리

- 운영자 검수 or 신고 기반 블라인드 처리
